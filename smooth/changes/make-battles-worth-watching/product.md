# 产品需求

## 背景

Orbit Wars 已经完成从创建舰队到真实对局、回放和 Agent 分析的基础闭环，但竞技信息仍偏技术档案：今日/本周榜单与长期总分排序混在一起，历史卡片突出回放工件信息，指挥中心生成 Key 后仍要求用户手工组合 Guide 与下一条指令。这会削弱排名信任、观战欲望和 Agent 接入效率。

## 目标

- 让今日、本周和历史榜单的排序含义真实、明确、可比较。
- 让公开历史首先回答“谁赢了、变化多少、为什么值得看”，技术验证信息退居次层。
- 让用户在生成 Agent Key 的唯一明文时刻，一次复制完整、安全、可执行的交接包。
- 保持统一舰队榜、Agent-only、中英双语和少量导航，不复制 Agentank 的品牌表达或外围经济系统。

## 核心体验

### 可信榜单

- 历史榜默认按舰队总分排序。
- 今日和本周默认按经过样本保护的周期胜率排序。
- 用户可切换总分、胜率和胜场；每个排序项明确显示口径。
- 榜单行展示周期比赛场次、胜、负、平和原始胜率。
- 低样本舰队不会仅凭一场胜利长期占据周期榜第一。
- Human/Agent 仍只是记录标识，不拆分为两套 rating。

### 观众化历史

- 每场卡片主层级展示胜负、双方舰队、rating delta、结束原因、时间和回放入口。
- 显示 0～100 的“战况强度”，由权威回放事件、时长和 rating 变化确定，并提供简短可解释信号。
- 展示最多三个关键事件，可直接进入回放对应 step。
- “精彩对局”只筛选达到明确强度阈值的比赛，不再等同于“存在事件”。
- schema、帧数、工件大小和保存时间折叠到“验证详情”。

### Command Center Agent 交接

- NEXT 卡片按实际状态变化：无有效 Key → 连接 Agent；刚生成 Key → 复制完整交接包；无当前 ready 策略 → 发布/选择策略；全部就绪 → 开战。
- Key 生成后提供“复制全部”和“仅复制 Key”。完整交接包包含 API Base、Agent Guide、Key、舰队 ID 和推荐首条指令。
- 明文 Key 仍只出现一次，交接包不保存到后端、不写入日志、不进入公共页面。
- 复制操作有中英文成功/失败反馈，并兼容受限内置浏览器。

## 范围

### 本轮包含

- 公开榜单排序参数、周期统计、样本保护和 UI 控件。
- 权威战况强度、高光事件、精彩筛选和历史卡片重排。
- Command Center 动态 NEXT、一次性 Agent 交接包和复制反馈。
- 对应 API、类型、单元测试、双语和响应式样式。

### 本轮不包含

- 版本级战绩对比和 replay → version 归因。
- 团队榜、Cups、成就、评论、每日战报、商店或钱包。
- Human 手动玩法和网页代码编辑器。
- 多地图、技能或引擎平衡调整。
- 生成式模型参与战况强度评分。

## 验收标准

- `/leaderboard?period=today` 与 `week` 默认按样本保护后的周期胜率排序，`all` 默认按总分排序。
- 总分、胜率、胜场三种排序均可通过 URL 保留并在中英文界面明确显示。
- 榜单胜负统计正确区分平局，不把平局计为失败。
- 每场公开比赛返回确定性的 `intensity` 和最多三个 `highlights`；同一数据重复计算结果一致。
- “精彩对局”仅返回强度达到阈值的比赛。
- 历史卡片无需展开即可看到胜负、双方、rating delta、结束原因、时间、强度和高光；工件信息默认折叠。
- 无有效 Key、刚生成 Key、无 ready 当前版本和全部就绪状态的 Command Center NEXT 均给出正确行动。
- 完整交接包包含必要字段且不被持久化；离开页面后无法重新显示 Key。
- 中英文桌面与窄屏无明显溢出、遮挡或未翻译主行动。
- lint、typecheck、全量测试、生产构建和浏览器验收通过。
