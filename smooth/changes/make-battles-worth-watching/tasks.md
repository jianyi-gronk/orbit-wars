# 任务

## 阶段 1：共享竞技事实

- [ ] **建立竞技统计纯函数** — 正确计算胜负平、原始胜率、样本保护胜率，并覆盖零场次和低样本测试。
- [ ] **建立战况强度与高光选择** — 基于权威事件、帧数和 rating delta 生成确定性强度、signals、featured 与最多三个高光。

## 阶段 2：可信榜单

- [ ] **扩展榜单 API 排序** — 增加 score/win_rate/wins、周期默认值、稳定排序和完整 record。
- [ ] **升级榜单 UI** — URL 保留排序条件，增加双语排序控件、完整战绩、胜率和样本保护说明。

## 阶段 3：观众化历史

- [ ] **扩展公开比赛摘要** — 返回 intensity、highlights，并让 featured 使用明确阈值。
- [ ] **重构历史卡片** — 强化胜负、双方、rating、原因、时间、强度和高光，折叠技术工件信息。
- [ ] **连接高光到回放 step** — 共享事件翻译，并支持 replay `?step=` 初始定位。

## 阶段 4：Agent 交接

- [ ] **抽取可靠剪贴板工具** — 复用 Clipboard timeout 与 legacy fallback，并迁移 ReplayPlayer。
- [ ] **生成安全 Agent 交接包** — 纯函数输出 API Base、Guide、Key、fleet ID 和首条指令，补充安全测试。
- [ ] **改造 Command Center NEXT** — 实现四种任务状态、一次性复制全部/仅 Key 和双语反馈。

## 阶段 5：验证与交付

- [ ] **补充质量保护** — 覆盖榜单语义、强度确定性、高光边界、交接包安全和关键视觉状态。
