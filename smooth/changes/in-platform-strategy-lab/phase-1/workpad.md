# 工作台

## 计划

- [x] 核对现有模板创建、ready 策略、比赛和候选模拟能力。
- [x] 定义无外部 Agent 的首次开战路径。
- [x] 定义站内策略实验室第一阶段范围与边界。
- [ ] 确认是否将生成式 AI 辅助改码纳入后续阶段。
- [ ] 进入技术设计，明确草稿、源码、沙盒模拟和发布接口。

## 验收标准

- 无 Agent Key 用户可以用 ready 模板直接完成训练赛。
- 站内支持调参和代码两种策略优化方式，草稿不会因切换视图丢失。
- 候选先模拟验证，只有通过后才能发布不可变版本。
- 外部 Agent 保留为可选高级入口。
- 新增界面与反馈支持中英文和窄屏。

## 验证

- 覆盖无 Key、ready 模板、草稿恢复、模拟失败、模拟成功、发布、恢复旧版本等状态。
- 验证模拟不改变 rating、公开战绩或当前版本。
- 验证沙盒错误脱敏、模板许可证与草稿隐私。
- 浏览器走通中英文“创建 → 训练 → 回放 → 优化 → 模拟 → 发布 → 再战”。

## 备注

- 当前创建舰队时已自动配置 `platform-basic` 或 `kaggle-structured-v11` 的 ready 版本。
- 当前 Agent Key 是自动化 API 的授权凭证，不是服务器执行 ready 策略的技术前提。
- 第一阶段推荐手动调参和代码编辑；生成式 AI 助手需要单独评估成本、滥用与配额。

## 疑问

- “平台内优化代码”第一阶段是否需要平台 AI 自动提出或应用修改，还是先交付用户手动调参和编辑代码的闭环？
- Kaggle 模板的站内可编辑与再发布范围需要在技术设计前完成许可证核验。
