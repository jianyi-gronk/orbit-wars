# 产品需求

本阶段属于“站内策略实验室”的第一阶段。整体目标是让没有外部 Agent 的用户也能完成完整游戏循环；后续阶段可再评估生成式 AI 辅助优化、模板社区和更复杂的调试工具。

## 背景

当前平台在创建舰队时已经自动绑定可运行的 `ready` 初始策略，并具备训练赛、沙盒验证、不可变策略版本和回放能力。但用户任务状态把“创建 Agent Key”放在开战之前，首页和指挥中心也持续强调外部 Agent 接入，导致用户将“手上已有 Agent”理解为参与游戏的前置条件。

外部 Agent 适合熟悉自动化和 API 的高级用户，不应成为新用户体验游戏、理解规则和迭代策略的门槛。

## 目标

- 新用户不创建 Agent Key，也能使用平台模板在三分钟内完成第一场训练赛。
- 用户可以在平台内查看、调整、模拟并发布自己的策略版本，形成可重复的优化闭环。
- 同时服务两种能力层级：新手通过可解释的策略参数开始，高级用户直接编辑策略代码。
- 外部 Agent 接入继续保留，但作为可选的自动化与高级工作流，而不是主流程门禁。
- 每次实际出战仍锁定不可变版本，保证排名、历史和回放可验证。

## 核心体验

### 双入口而非 Agent 门禁

创建舰队时继续选择初始模板。创建成功后，用户直接获得两个清晰选择：

1. **立即训练**：使用当前 `ready` 模板匹配一场不计分比赛，推荐给首次进入的用户。
2. **优化策略**：进入站内策略实验室，调整策略后再训练。

“连接外部 Agent”收进次级入口和任务菜单。只要舰队存在且当前策略为 `ready`，页头主任务就是“立即开战”或“继续优化”，不能再因为缺少 Agent Key 阻止开战。

### 策略实验室

策略实验室围绕当前舰队和一个可编辑草稿工作，提供两种可随时切换的视图：

- **策略调参**：用可解释控件调整扩张倾向、进攻阈值、驻防比例、目标优先级等平台明确支持的参数。每项说明它会影响什么，不要求用户读代码。
- **代码模式**：查看并编辑完整的策略入口代码，显示可用 API、运行限制、模板来源和必要归属信息。用户可以从当前版本复制草稿，也可以重置为任一合法模板。

两种视图必须操作同一份草稿；能可靠互相映射的字段保持同步，无法映射的自定义代码不得被调参视图静默覆盖。

### 安全试错闭环

用户编辑草稿后可以执行“运行模拟”：

1. 平台先进行格式、接口和安全校验。
2. 候选策略在隔离环境中对固定基准场景和一个训练对手运行。
3. 页面显示通过/失败、错误位置、胜负、关键指标和可查看回放。
4. 模拟候选不会改变当前策略、积分或公开战绩。
5. 只有验证通过的候选才能发布为新的不可变策略版本。

发布后，用户可将新版本设为当前版本并立即开始训练；旧版本继续保留，可查看、对比和一键恢复。真实回放结束页提供“优化这版策略”入口，把失败或高光带回策略实验室。

## 用户流程

```text
登录 → 创建舰队并选择模板 → 立即训练 → 查看回放
                                  ↓
                              优化策略
                     调参 / 编辑代码 → 运行模拟
                                  ↓
                       发布不可变版本 → 再战

任务菜单 → 连接外部 Agent（可选高级入口）
```

## 范围

### 本阶段包含

- 移除 Agent Key 对首次开战和页头主任务的产品门禁。
- 新建站内策略实验室及中英文入口。
- 当前版本只读查看、创建草稿、草稿持久化、重置模板和未保存提示。
- 新手调参视图与高级代码视图。
- 候选策略校验、沙盒模拟、结果摘要和回放入口。
- 验证通过后发布不可变版本、设为当前版本和恢复旧版本。
- 从创建完成页、指挥中心、回放结果和任务菜单进入实验室。
- 桌面、窄屏、键盘操作和 reduced-motion 状态。

### 本阶段不包含

- Human 实时手动操作。
- 平台自动调用生成式 AI 改写代码。
- 安装任意第三方依赖、开放系统命令或访问公网。
- 多人协作编辑、模板市场、社区代码分享和付费算力。
- 让未验证草稿参加排位或覆盖既有不可变版本。
- 对来源或许可证不允许修改的第三方模板提供可编辑副本。

## 内容与安全边界

- 平台必须明确区分“草稿”“模拟候选”“已发布版本”“当前出战版本”。
- 草稿默认私有；未经用户明确发布，不进入公开档案或比赛工件。
- 模拟错误不得泄露宿主机路径、环境变量、密钥或其他用户数据。
- 模板必须标注来源和许可证；第三方模板只有在允许站内复制与修改时才可进入编辑器。
- 发布动作必须显示版本说明、验证结果和将要锁定的内容摘要。
- 站内界面、教程、错误和操作反馈支持简体中文与英文；代码接口标识保持一致，不做破坏性的本地化。

## 验收标准

- 一个没有 Agent Key 的新账户可以创建模板舰队、进入竞技场并完成训练赛，流程中不出现“必须连接 Agent”的阻断。
- 舰队存在且当前版本为 `ready` 时，页头主行动不依赖 Agent Key；外部 Agent 入口仍可从任务菜单和指挥中心到达。
- 用户可以从当前版本或合法模板创建私有草稿，刷新或重新登录后草稿仍可恢复。
- 用户可以在调参和代码模式间切换；平台不会因切换视图静默丢失或覆盖自定义代码。
- 微小语法错误、接口错误、超时和安全违规都有可定位、可理解且不泄密的中英文反馈。
- 运行模拟不会改变 rating、公开战绩或当前出战版本，并至少提供结果摘要及对应回放入口。
- 未通过验证的候选不能发布；通过后会生成新的不可变版本，且不会覆写旧版本。
- 用户可以选择新版本出战，也可以恢复任一仍为 `ready` 的旧版本。
- 真实回放结束后可以一键回到对应策略版本的实验室草稿。
- `/zh` 与 `/en` 在桌面和窄屏均无关键操作缺失、明显溢出或未翻译主文案。

## 成功指标

- 新账户从登录到首场训练赛的完成率提高。
- “创建舰队后、首次开战前”的流失率下降。
- 无 Agent Key 用户的训练赛占比、策略模拟次数和第二版本发布率可被统计。
- 外部 Agent 用户的既有 API 流程和比赛成功率不下降。
