# 踩坑记录

## Canvas 上层装饰破坏了太阳完整性

- 症状：太阳代码本身是完整圆盘，但实际回放截图仍有一条竖线穿过太阳，圆盘也因纯色和硬描边显得像占位元素。
- 根因：阵营中轴属于 Canvas 上方的 CSS overlay，单独检查 Pixi 渲染代码无法发现；首轮验收也只确认了“元素存在和装饰删除”，没有继续评估色彩层次。
- 如何被发现：生产构建后刷新真实回放并截图，与 Kaggle 参考直接比较。
- 修复 / 预防：中轴在太阳直径范围内渐隐；实体使用同色相轻微径向渐变并移除硬描边。以后验收 Canvas 视觉时必须同时检查所有上层 overlay。
- 这能改进 harness 吗：是 — UI 验收应包含真实叠层后的截图，而不是只检查组件源码。
