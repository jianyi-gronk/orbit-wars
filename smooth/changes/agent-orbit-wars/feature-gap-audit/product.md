# 产品需求

## 产品目标

把 Orbit/Wars 从“后端能力完整、Web 主要为高保真样机”推进为可持续使用的真实中英双语产品闭环。用户必须能登录、创建并管理唯一舰队、生成和撤销 Agent Key、查看与切换真实策略版本、发起训练或排位、查看统一榜单和公开档案，并通过永久链接播放该场比赛的真实回放。

本变更以 2026-07-18 的 AgenTank 核心 1v1 Agent 竞技闭环为功能基线，但不复制其品牌、文案、视觉资产或坦克机制。当前公开产品突出轨道战略、Agent 自主竞技和可验证航迹；Human 实时指挥只保留底层协议、战术台与功能开关，不作为当前可选玩法开放。

## 核心用户场景

- 新用户通过 OIDC 登录后建立唯一舰队；刷新页面后数据仍存在。
- 新用户创建舰队时可在平台轻量策略与经审计的公开 Kaggle starter 之间选择，所选模板成为首个 ready 策略版本。
- 空环境可幂等初始化一组系统 Agent 舰队并安排真实排位预热赛，使新用户首次进入时已有多个可匹配对手和统一榜单记录。
- 舰队拥有者在指挥中心查看真实 rating、版本、Key 与近期比赛，所有操作直接写入 API。
- 用户看到统一积分对应的本地化竞技段位和段内积分，例如总积分 625 显示为“黄金 III · 25 分”；底层唯一 rating 与全服名次不变。
- 用户在竞技场获取真实推荐对手并创建 Agent 训练或排位比赛；当前 Web 不提供 Human 控制选择。
- 匿名访客按今日、本周、历史和控制标签查看统一榜单，并进入真实舰队档案、对局历史和回放。
- 回放播放器按 `publicId` 加载权威 checkpoint/delta 帧、关键事件、参与者、结果和事实型分析，不再使用演示常量。
- 外部 Agent 可依据完整 Guide 完成读取上下文、候选模拟、发布、挑战、读取比赛和紧凑回放分析。
- 上述公开与登录后流程均支持简体中文和英文；语言切换保留业务路径、资源 ID 和筛选条件。

## 功能范围

### 本变更包含

1. OIDC 登录、回调、退出、会话状态和本地开发专用身份通道。
2. 当前账号舰队查询，以及真实舰队创建、Key 生命周期、版本列表/切换和近期战绩接线。
3. 真实匹配推荐、训练/排位创建和比赛票据交接。
4. 真实统一榜单、公开舰队档案和公开对局历史。
5. 权威回放播放器，以及供 Agent 消费的 compact summary/events JSON。
6. Agent 候选策略模拟：候选包须经过同等验证与 Sandbox 边界，不成为当前版本，也不污染不可变发布历史。
7. `zh-CN` 与 `en` 同等完整的语言 URL、切换、偏好、系统文案、SEO 和无障碍文案。
8. 双语 Agent Guide、About、Q&A、Updates、Privacy、Terms 与联系入口。

### 明确后置

- Human 手动实时指挥的公开入口；底层同协议实现继续保留，后续通过功能开关开放。
- 团队榜、团队管理、杯赛、Raid 和多人自由混战。
- TankBook、Agent 墙帖、评论与社区审核系统。
- 商店、成就、邀请奖励、钱包、链上铸造和 Codex pet。
- 视频文件导出；本阶段提供永久网页回放、原始 artifact 和 Agent compact JSON。

## 双语与协议边界

- 公开页面使用 `/zh/...` 与 `/en/...`；无语言 URL 首次按显式偏好、浏览器语言、简体中文默认值依次跳转。
- 用户原创内容保持原文。API 路径、字段、enum、scope、错误码和回放事件类型保持稳定英文。
- 当前公开比赛由 Agent 控制；未来开放 Human 时仍共享数据、匹配池和唯一 rating，语言及控制筛选均不得产生独立榜单。

## 验收标准

1. 中文和英文均能从登录完成舰队创建，并从 API 读回同一个 `publicId`。
2. 指挥中心显示数据库中的舰队、rating、版本和 Key；生成、撤销、切换后刷新仍一致。
3. 竞技场从真实 offer 创建比赛，响应包含真实 match ID、票据、控制类型和模式。
4. 榜单、档案与历史不包含页面内硬编码舰队/战绩；筛选只改变同一榜单视图。
5. 任意真实 replay `publicId` 的播放器帧、事件、参与者和结果均来自该 replay；不存在固定演示替代。
6. compact replay 接口返回稳定摘要、关键事件、版本归因和深层回放链接。
7. 候选策略模拟在验证失败时拒绝执行，成功时不创建公开策略版本或移动当前版本指针。
8. 任意产品页可中英切换且保留路径、ID、筛选；两种语言没有功能缩水或未翻译核心系统文案。
9. Agent Guide 的两种语言覆盖所有 scope、关键端点、幂等、限流、错误恢复和完整迭代示例。
10. lint、typecheck、单元/集成测试、Next 生产构建和关键双语用户旅程验证通过。
11. 默认配置下所有 Web 开战入口只创建 `controllerType=agent`；Human 选择和实时战术台入口不可见，但设置显式功能开关后现有 Human 路径仍可恢复。
12. Agent-only 入口使用紧凑的只读状态表达，不把原双选卡片拉伸为单张巨型卡；中英文窄屏的对手名称、匹配摘要和主按钮均不溢出。
13. 创建舰队默认使用平台基础模板，也可选择公开 Kaggle Structured Baseline；模板来源可追溯，未知模板被拒绝，创建后当前 ready 版本与用户选择一致。
14. 预热命令幂等创建 6 个系统 Agent（平台/Kaggle 模板各 3 个）和 6 场真实 Agent 排位赛；重复执行不重复创建或排队，完成的排位赛按现有统一 rating 规则结算。
15. 榜单、舰队档案和指挥中心将统一总积分确定性映射为青铜/白银/黄金/铂金/钻石 III–I 以及大师段位，并以中英文显示段位、段内积分、总积分和全服名次。
16. 对局历史参考 Kaggle episode 的“索引 + 每场不可变工件”形式，明确显示回放帧数、工件大小和保存时间；公开回放瞬时读取失败后可直接重试，不得停留在无法恢复的通用错误页。
