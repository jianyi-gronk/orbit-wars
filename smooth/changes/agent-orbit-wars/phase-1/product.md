# 产品需求

## 产品概述

构建一个支持人类与 AI 双控制方式的星际舰队对战网站。玩家既可以亲自下达舰队指令，也可以作为教练把策略权限交给 AI Agent；两种方式共享同一套战场规则、舰队身份、版本记录和战斗回放。

第一阶段交付两个互通的 2P 闭环：人类玩家可以创建舰队后直接进入手动战斗；AI 玩家可以接入 Agent、模拟策略、发布版本并发起排位战。所有对局都可以观看回放并用于下一轮学习或策略迭代。

产品采用原创太空战争世界观和视觉资产，不直接使用《Star Wars》的名称、Logo、角色、飞船造型、音乐或剧情元素。对外定位使用“星际战争”“轨道战争”或独立品牌名。

## 背景与机会

现有 `orbit-wars` 项目已经具备可运行的星球、舰队、轨道运动和策略 Agent 对战基础，但目前主要面向本地研发、Kaggle 评测和回放诊断，普通玩家难以理解、参与或分享。

参考 AgenTank 的有效产品机制：

- 人类用户承担拥有者、教练和策略提出者的角色，AI Agent 承担策略编写与迭代。
- 规则保持容易理解，同时允许策略产生足够大的表现差异。
- 模拟用于低成本试错，公开比赛用于形成永久战绩、排名和可分享回放。
- Agent Guide 与专属密钥让外部 AI 可以读取舰队上下文、测试策略和发布新版本。
- 排行榜、版本记录和逐帧回放把一次性小游戏变成持续竞争社区。

## 要解决的问题

1. 策略对战项目的进入门槛高，普通用户看不懂代码、评测脚本和本地回放。
2. AI 生成策略缺少清晰的授权、测试、发布和复盘链路。
3. 单次战斗缺少长期价值，没有身份、版本谱系、公开战绩和竞争目标。
4. 轨道运动和舰队决策很难仅靠结果数字理解，需要可视化回放说明“为什么赢或输”。

## 目标用户

### 核心用户

- 对 AI Agent、策略编程和自动对战感兴趣的开发者。
- 喜欢太空战争、观战、排行和养成，但不希望先学习编程的玩家。
- 希望展示 Agent 能力、模型风格或提示词成果的 AI 创作者。

### 次级用户

- 只观看高质量比赛、排行榜和精彩回放的观众。
- 希望举办主题挑战或邀请赛的社区组织者。

## 产品原则

1. **双控制入口**：创建舰队后可以选择“亲自指挥”或“交给 Agent”，不要求用户必须懂代码或必须使用 AI。
2. **观战可解释**：每次结果都应能从回放、事件和关键指标中找到证据。
3. **模拟与正式战分离**：模拟不影响排名；正式战形成永久记录并更新排名。
4. **规则简单、策略有深度**：玩家几分钟内理解胜负目标，但不同策略能形成明显战斗风格。
5. **版本可追溯**：每次发布都形成独立策略版本、说明和对应战绩。
6. **原创品牌**：营造电影级太空战争氛围，但不依赖第三方影视 IP。

## 视觉与氛围

### 设计定位

整体采用“原创太空歌剧 + 复古未来军事终端 + 高端时尚编辑设计”风格。目标是让用户感受到史诗级星际冲突、巨大尺度和前线指挥压力，同时以更大胆的版式建立品牌个性，避免落入常见游戏门户的卡片堆叠模板。

### 布局方向

- 首页、舰队档案和赛季专题可以借鉴高端时尚杂志与品牌型录的编辑式构图方法，但不复刻任何具体网站。
- 使用超大标题、非对称网格、跨栏排版、纵向编号、局部文字压图和有节奏的留白。
- 在满版电影感视觉与克制的信息页面之间形成强烈节奏变化，不要求所有区块都对齐成规整卡片。
- 排行榜可以呈现为“舰队名册/赛季型录”，舰队档案可以呈现为“指挥官封面故事”，精彩回放可以呈现为“战役专题”。
- 鼓励用轨道弧线穿越不同内容区、让排行数字成为构图元素、让战斗数据形成装饰性信息纹理。
- 动效可以具有镜头推进、扫描揭示、排版错位复位和轨道转场，但必须支持减弱动态效果。
- 创意自由主要用于品牌页、观战页和档案页；实时战斗操作区采用稳定布局，不能为了视觉冲击牺牲指令速度和状态辨识。

### 可采用的类型特征

- 深黑宇宙背景、星云、行星逆光和大尺度舰队剪影。
- 象牙白、旧金属灰、警戒红、能量蓝与琥珀黄组成的高对比色彩系统。
- 磨损金属、铆钉、机械面板、雷达噪点和低清晰度全息投影质感。
- 战术星图、轨道线、舰队航迹、扫描扇区和克制的发光反馈。
- 电影级宽银幕构图、低沉克制的文案和“前线指挥部”仪式感。
- 标题使用原创的几何无衬线或工业窄体字，正文保持高可读性。
- 音效强调机械继电器、通讯噪声、舰船引擎和空间战斗冲击，但形成独立声音识别。
- 允许高反差的字体尺寸、图片裁切和版面错位，形成时尚编辑感，而不是传统科幻 HUD 全屏覆盖。

### 明确避免

- 不使用或近似重绘《Star Wars》的 Logo、片头透视滚屏和标志性标题排版。
- 不复刻光剑、绝地、原力、帝国/反抗军符号、风暴兵或已知角色形象。
- 不复刻千年隼、X-wing、TIE fighter、歼星舰等可识别飞船轮廓。
- 不模仿具体电影海报、经典镜头、配乐旋律、角色台词或 UI 画面。
- 不把参考站 AgenTank 的页面布局、组件造型、文案和品牌色直接迁移过来。
- 不照搬具体时尚品牌或杂志网站的首页结构、标志性动效、摄影作品和字体组合。

### 独立识别元素

- 以“轨道”作为核心视觉母题：导航、按钮、加载、排名进度和回放时间线都使用圆弧、引力轨迹和交汇点语言。
- 舰队以抽象楔形、环形推进器和模块化几何结构建立原创轮廓体系。
- 每个阵营使用原创星图纹章和航迹编码，而不是沿用影视作品中的阵营符号。
- 战斗界面强调可读的策略信息，视觉效果不得遮挡星球归属、兵力、轨迹和操作反馈。

## 核心玩法

### 玩家身份

玩家创建一个“舰队指挥部”，包括：

- 舰队名称
- 指挥官代号
- 阵营宣言
- 原创视觉风格
- 初始策略倾向：扩张型、突袭型、防守型或均衡型

这些设置建立公开身份，但不直接决定数值强弱。

每场战斗开始前必须明确选择控制方式：

- **人类指挥**：玩家在有限的命令时间内选择源星、发射方向和舰船数量。
- **Agent 指挥**：已发布的策略版本自动接收战场状态并输出指令。

正式排位战开始后不得在中途切换控制者，确保战绩归因清晰。公开战绩和回放必须显示本场由人类还是哪个 Agent 策略版本控制。

### 战场

- 第一阶段为 1v1 星系争夺战。
- 地图由静止或沿轨道运动的星球、初始母星和中立星球组成。
- 星球持续生产舰船；舰队从己方星球发射，沿给定方向飞行，并与首先接触的星球发生战斗。
- 玩家策略需要决定何时扩张、保留多少守军、如何增援、是否攻击敌方以及如何利用轨道位置。

### 胜负

- 消灭对手全部有效星球和舰队时立即获胜。
- 达到回合上限时，按公开且稳定的综合战力规则判定胜负；评分至少反映星球控制、生产能力与舰船力量。
- 回放必须明确显示胜负原因，避免只展示一个不透明分数。

### 核心循环

```text
                         ┌─> 亲自指挥 ─> 手动训练/匹配 ─┐
创建舰队 ─> 选择控制方式 ┤                            ├─> 回放复盘 ─> 再次挑战
                         └─> 交给 Agent ─> 模拟/发布 ──┘
```

人类操作和 Agent 操作共享同一个匹配池、排名分数和排行榜。控制方式不提供额外加成或独立计分，只在对局、战绩和排行榜中作为公开标签。为保证同榜公平，两种控制方式接收相同的战场信息，遵守相同的指令格式、行动频率和行动时限。

## 关键产品模块

### 1. 首页

- 用一屏说明“创建舰队，让 AI 替你征服星系”。
- 直接展示正在进行或近期精彩战斗，避免首页只是静态营销页。
- 清晰解释创建、交给 Agent、观看并迭代三个步骤。
- 提供“创建舰队”“观看战斗”“查看 Agent Guide”三个主要入口。

### 2. 舰队创建

- 用户无需写代码即可创建第一支舰队。
- 通过少量文本和策略倾向完成创建。
- 创建后立即提供可运行的基础策略，使用户能马上完成第一次模拟。

### 3. 指挥中心

- 展示当前策略版本、排名、近期战绩和舰队状态。
- 提供 Agent 接入说明与可撤销/轮换的舰队密钥。
- 展示策略版本历史、版本说明和各版本战绩。
- 支持将历史稳定版本重新设为当前版本。

### 4. 模拟训练

- 用户或 Agent 可选择训练对手和地图发起模拟。
- 模拟不影响公开排名，也不进入正式战绩。
- 结果提供回放、关键事件和基础诊断信息。
- 模拟频率需要受到合理限制，避免无限资源消耗和刷测试。

### 5. 人类指挥

- 玩家点击己方星球作为出发点，选择或拖拽发射方向，并设定本次派出的舰船数量。
- 发令界面提供可理解的方向、预计航线和兵力反馈，但不替玩家自动判断最优行动。
- 每个命令周期有明确倒计时；未提交指令时视为本周期不行动。
- 支持对训练机器人、人类玩家或指定 Agent 进行手动战斗。
- 手动训练不影响排名；手动排位与 Agent 排位进入同一个排行榜。

### 6. 竞技场

- 用户选择本场由人类还是 Agent 控制，再从同一个合理实力区间内匹配或挑战公开舰队。
- 正式战不可撤销，生成永久战绩、排名变化和公开回放。
- 对手选择应兼顾公平性和多样性，限制重复刷同一弱对手。
- 人类对人类、Agent 对 Agent 和人类对 Agent 的正式比赛均使用同一套计分规则并影响统一排名。

### 7. 回放中心

- 可暂停、继续、调速、逐回合前进和拖动时间轴。
- 清晰区分双方颜色、星球归属、舰队轨迹和交战结果。
- 标出关键事件：首次占领、最大舰队出击、母星易手、领先反转和最终胜因。
- 同步展示星球数、总产能、驻守舰船、在途舰船和控制率曲线。
- 提供可分享链接，并允许把回放链接交给 AI 作为下一轮分析输入。
- 显示双方控制方式；人类操作记录命令时间，Agent 操作记录策略版本。

### 8. 排行榜与公开档案

- 使用一个统一排行榜，展示段位、胜负记录、近期变化和当前常用控制方式。
- 每条排名和战绩显示“人类指挥”或“Agent 指挥”标签，并支持按控制方式筛选观察，但筛选不会形成另一套排名。
- 支持今日、本周和历史榜单。
- 舰队公开页展示身份、版本谱系、近期比赛和代表性回放。

### 9. Agent Guide

- 用外部 AI 易读的方式描述规则、策略生命周期、权限边界和常见错误。
- 舰队密钥只授权一支舰队的读取、模拟、版本发布和挑战能力。
- 用户可以随时轮换或撤销密钥。
- Agent 的每次发布和正式挑战都要留下来源与版本说明。

## 第一阶段范围

### 包含

- 原创品牌与完整首页体验。
- 账号与单舰队创建。
- 一套可立即运行的基础策略。
- 人类指挥界面、训练机器人和统一排位入口。
- 2P 模拟战和正式排位战。
- Agent Guide、单舰队密钥和策略版本发布。
- 舰队指挥中心。
- 可交互逐帧回放及关键战斗指标。
- 基础段位、排行榜和公开舰队档案。
- 桌面端完整体验；移动端支持首页、排行榜和回放观战。

### 不包含

- 4P 自由混战。
- 排位战中途在人类与 Agent 之间切换控制权。
- 公会、组队赛、赛季杯赛和竞猜。
- 付费卡牌、数值养成、NFT 或链上资产。
- 用户自定义地图编辑器。
- 复杂社交动态、评论和私信。
- 直接使用《Star Wars》或其他影视作品的授权内容。

## 已确认的产品决策

- 第一阶段是可公开访问的真实 2P 平台，不是只在本地运行的演示站。
- 人类与 Agent 共用同一个匹配池、排名和计分规则，只标记控制方式。
- 正式排位赛开始后不允许中途切换控制者；训练模式接管后续再评估。
- 第一阶段复用并固定现有 Orbit Wars 核心规则与 `agent(obs)` 动作协议，在其上增加网站控制、版本、沙箱和回放层。
- “星球大战”只表示类型氛围；品牌、舰船、阵营、字体、声音和界面资产全部原创。

## 后续阶段草图

- **第二阶段**：4P 自由混战、赛季、锦标赛、地图池和更丰富的策略档案。
- **第三阶段**：舰队外观生成、社区战报、Agent 自动复盘和跨版本对比。
- **第四阶段**：团队联赛、创作者活动及经过验证的商业化机制。

## 验收标准

1. 新访客在首页首屏可以理解舰队既能由人亲自指挥，也能交给 AI Agent。
2. 新用户无需写代码即可创建舰队，并在一次连续流程中完成首场手动训练或 Agent 模拟。
3. 用户可以获得清晰的 Agent Guide 和专属舰队密钥，并能独立撤销或轮换密钥。
4. Agent 可以围绕同一舰队形成可追溯的策略版本；每个版本包含说明、时间和来源。
5. 模拟和手动训练不改变排名；正式战会生成永久记录、明确的排名变化和公开回放。
6. 对局结束后，用户能从回放中识别至少五类关键事件，并理解最终胜因。
7. 人类与 Agent 使用同一排名口径；排行榜、舰队档案、战绩和回放均准确标记每场比赛的控制方式。
8. 用户可以分享一个无需登录即可观看的公开回放链接。
9. 第一阶段所有公开名称、文案和视觉资产均为原创，不依赖第三方影视 IP。
10. 桌面端可以完成手动操作与 Agent 管理两套核心循环；移动端可以顺畅完成观战和查看排名。
11. 正式对局开始后不能切换控制者，回放可以准确归因到人类玩家或具体 Agent 策略版本。
12. 未接触项目背景的测试者能将视觉描述为“太空歌剧/星际战争”，但不能把任何核心界面、舰船或标志误认为《Star Wars》官方内容或 AgenTank 页面复刻。
13. 首页、档案和观战页面具有明显的编辑式节奏与原创构图，而实时战斗页面仍能让玩家快速识别可操作星球、兵力、归属、航迹和倒计时。
14. 关闭或减弱动态效果后，所有核心信息和操作仍可完整使用。

## 成功指标

- 创建舰队的用户中，完成首场模拟的比例。
- 创建舰队的用户中，完成首场手动训练的比例。
- 人类指挥用户转化为 Agent 指挥用户的比例，以及反向转化比例。
- 完成首场模拟的用户中，发布第二个策略版本的比例。
- 每支活跃舰队每周产生的有效策略版本数。
- 正式比赛后打开回放的比例及平均观看进度。
- 回放分享带来的新访客和新舰队创建数。
- 排位对局的有效完成率、崩溃率和重复对手占比。
