# 产品需求

## 产品概述

Orbit/Wars 全部公开产品体验需要同时支持简体中文和英文。语言能力不是首页局部翻译，而是覆盖从首次访问、舰队创建、Human/Agent 战斗、指挥中心、排行榜、公开档案、回放到 Agent Guide 的完整闭环。

双语版本必须保持相同的玩法、权限、安全边界和信息完整度；任何语言都不能成为功能缩水版。用户创建的舰队名称、宣言、策略版本说明等原创内容按原文展示，不进行未经作者确认的机器翻译。

## 用户场景

- 中文用户首次访问时直接看到完整中文体验，也可随时切换英文。
- 英文用户通过搜索、分享链接或直接访问进入英文页面，不需要先理解中文导航。
- 用户分享排行榜、舰队档案和回放链接时，链接可以明确携带语言版本；接收者仍可自行切换。
- Human 与 Agent 对局中的倒计时、操作提示、错误反馈、胜负原因和回放事件均随界面语言变化。
- 外部 Agent 继续消费稳定、语言无关的协议字段和错误码；Agent Guide 分别提供中英文说明。

## 产品机制

### 语言入口

- 支持 `zh-CN` 与 `en` 两种语言。
- 首次访问优先使用明确的语言 URL；无语言 URL 时参考浏览器语言，无法判断时默认简体中文。
- 全站导航提供可键盘操作且有明确名称的语言切换入口。
- 切换语言后保留当前页面、查询条件和公开资源 ID，并记住用户偏好。
- 公开页面使用可分享、可索引的语言 URL，并提供正确的 canonical 与 alternate language 信息。

### 翻译范围

- 翻译所有系统文案：导航、标题、按钮、表单标签、帮助、模式说明、确认提示、空状态、加载状态和错误信息。
- 翻译所有游戏系统文案：Human/Agent 标签说明、训练/排位边界、命令反馈、断线/超时原因、关键事件、事实型胜因和 rating 变化说明。
- 翻译 SEO metadata、无障碍名称、ARIA live 消息和 reduced-motion/低性能说明。
- 时间、数字、排名、复数和日期使用当前语言的格式规则。
- 舰队名、指挥官代号、宣言、外观描述、策略说明和用户提交内容保持原文，并用正确的语言标记与排版承载。

### 稳定协议边界

- API 路径、JSON 字段、enum、scope、错误码、策略 manifest 和 Agent 协议保持英文且稳定，不随界面语言改变。
- 界面根据稳定错误码显示本地化说明；调试视图可同时保留原始错误码。
- 回放原始事件类型保持稳定标识，播放器只本地化显示名称和说明。

## 范围

### 本阶段包含

- 首页、创建流程、竞技场、战术台、指挥中心、榜单、舰队档案、回放和 Agent Guide 全量双语。
- 语言检测、显式切换、偏好持久化、语言 URL 和分享链接保留。
- 中英文 SEO metadata、无障碍文案、日期/数字格式和错误码映射。
- 翻译完整性检查，防止新增页面或文案只提供单一语言。
- 现有中文 URL 的兼容跳转，不破坏公开回放与档案链接。

### 本阶段不包含

- 自动翻译用户创建的内容。
- 简体中文和英文以外的第三种语言。
- 按语言拆分匹配池、排名、战绩或用户身份。
- 为不同语言改变规则、数值、功能或视觉品牌。

## 验收标准

1. 用户可以从任意产品页面在中文和英文之间切换，并停留在同一业务页面与资源上。
2. 首次访问根据语言 URL/浏览器偏好进入合理语言，后续访问记住显式选择。
3. 首页到首场 Human 训练或 Agent 模拟的完整流程在两种语言下都不存在未翻译系统文案。
4. 战术台的倒计时、命令、服务器反馈、键盘帮助、错误和无障碍播报均有中英文版本。
5. 排行榜筛选不影响统一 rating；语言切换不改变筛选条件、资源 ID 或业务数据。
6. 舰队档案和回放链接可携带语言并可匿名分享，切换语言不创建另一份战绩或回放。
7. Agent Guide 的中英文内容表达同一协议，代码示例、scope、字段与错误码保持一致。
8. API 和回放中的稳定代码不翻译，所有面向人的解释由界面本地化。
9. 用户原创内容保持原文，不被自动改写；中英文混排不造成截断、溢出或不可读。
10. 自动检查可以发现任一语言缺 key、直接写死的公开系统文案和错误码未映射问题。
11. 中英文页面具备正确的页面语言、metadata、canonical 和 alternate language 标记。
12. 桌面与移动端、键盘操作、reduced motion 和低性能模式在两种语言下均保持可用。

## 已确认决策

- 支持简体中文和英文，两者为同等完整的一等语言。
- 公开页面采用明确语言 URL，以兼顾分享、SEO 和可预测导航。
- 浏览器语言仅用于首次兜底，显式用户选择优先并持久化。
- 用户内容保持原文；系统 UI、状态和解释全部本地化。
- Human 与 Agent 继续共享同一业务数据、匹配池和统一排行榜，语言只影响展示。
