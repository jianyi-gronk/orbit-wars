# 产品需求

## 背景与目标

Orbit/Wars 已具备真实的 Agent 创建、比赛、榜单和回放闭环，但公共首页仍以编辑型长页面呈现，一级导航同时暴露竞技场、榜单、历史和指挥中心，整体更像产品后台而不是正在运行的星际竞技游戏。

本变更将公共体验升级为“舰桥任务序列”：用更明确的战术 HUD、轨道雷达、星图纵深和能量反馈强化原创太空战争气质；首页按全屏场景推进，让用户通过滚轮、触控或键盘看到上一幕退场、下一幕登场；同时收敛一级导航，让开战成为最明显的行动。

参考 Agent-first 对战产品的单一主行动、三步玩法和分段叙事节奏，但不复制其坦克机制、品牌、文案、装甲资产或页面造型。

## 核心体验

- 用户打开首页后首先进入完整首屏战情简报，能立即理解“建立舰队、交给 Agent、开战并进化”的主循环，并看到唯一主行动。
- 桌面用户滚动鼠标滚轮时，一次进入相邻场景；当前场景向后淡出并缩小，下一场景从纵深和下方进入。
- 触控用户使用自然纵向滑动；键盘用户使用方向键、PageUp/PageDown、Home/End 在场景间移动。
- 首页持续显示当前场景编号、总场景数和下一步提示，用户不会被滚动接管困住。
- 任意按钮或链接点击时出现短促、克制的轨道能量反馈，且不会延迟导航或业务提交。
- 顶栏一级导航只保留竞技场、统一排名和指挥中心；历史、Agent Guide 与说明页进入紧凑任务菜单，登录/开战与语言动作保留。
- 内页沿用同一套舰桥边框、网格、扫描线和按钮反馈，增强整体游戏一致性，但不牺牲表单、榜单和回放可读性。

## 首页场景

1. **部署简报**：品牌主张、Agent-only 说明、立即开始与观看战斗。
2. **作战循环**：创建舰队、部署 Agent、复盘进化三步流程。
3. **竞技网络**：真实预热 Agent、统一段位、公开比赛与永久回放入口。
4. **接入协议**：Agent Guide、策略版本和最终开战行动。

## 范围

### 包含

- 重构公共首页为四幕全屏场景，并提供滚轮、触控、键盘、指示器和可跳转控制。
- 精简顶栏一级导航并增加可访问的任务菜单。
- 增加全局点击能量反馈、按钮按压反馈和首页轻量指针视差。
- 强化全站原创轨道战争视觉语言，包括舰桥框架、星图网格、扫描和状态标记。
- 中英文内容与交互完全对齐。
- 响应 `prefers-reduced-motion`，关闭滚轮接管、视差和装饰性过渡。

### 不包含

- 复制 AgenTank 的品牌、坦克角色、卡通资产或具体页面布局。
- 添加背景音乐、点击音效、震动或需要用户授权的媒体能力。
- 改动比赛规则、API、积分、匹配或 Agent 执行逻辑。
- 将所有内页强制改为全屏滚动场景。

## 验收标准

1. 顶栏桌面一级导航最多三个业务入口，历史与 Guide 可在任务菜单内到达；移动端无横向溢出。
2. 首页包含四个可定位场景，桌面滚轮一次只推进一幕，并呈现前一幕退场与后一幕登场。
3. 触控自然滑动、键盘导航和场景指示器均可使用，不存在无法返回上一幕或跳出页面的滚动陷阱。
4. 业务按钮和链接点击产生可见但短于 700ms 的能量反馈，导航与提交不会被阻塞。
5. 中英文首页拥有相同场景、主行动、菜单与交互能力，动态长文本不溢出。
6. `prefers-reduced-motion: reduce` 下不执行滚轮接管、指针视差、场景位移或点击粒子动画。
7. 首页与关键内页在桌面和 390px 窄屏均无页面横向溢出，核心 CTA 可键盘聚焦。
8. Web lint、typecheck、测试和 Next.js 生产构建通过。
