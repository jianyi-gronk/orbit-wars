# 踩坑记录

## ReplayFrame 结构变化后开发热更新保留旧帧

- 症状：代码已接入 `fleets`，但热更新后的现有回放仍只显示星球，并出现 useEffect 依赖数组长度变化警告。
- 根因：React Fast Refresh 保留了更新前加载的 `ReplayFrame[]` 状态；旧对象没有 `fleets` 字段，同时组件 hook 依赖数组结构发生变化。
- 如何被发现：检查 STEP 83 截图和浏览器运行时日志时发现，权威 API 同一帧实际包含 43 支舰队。
- 修复 / 预防：回放数据结构或 hook 依赖结构变化后，使用干净页面加载或生产模式验证，不把 Fast Refresh 旧实例当作功能结果。
- 这能改进 harness 吗：是 — UI 验证流程应在数据模型变化后强制全页干净加载。
