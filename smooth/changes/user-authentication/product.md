# 产品需求

## 背景

Orbit Wars 当前公开预览中的所有访问者共享固定预览身份，用户无法注册自己的账户、保存独立舰队和战绩，也无法在不同设备继续自己的进度。项目虽然已有 OIDC 登录骨架，但没有可用的注册界面、密码登录、找回密码、账户入口和真实多用户隔离。

## 目标

- 让拥有 GitHub 账号的新用户无需预先拥有 Agent 即可自动注册并建立自己的舰队。
- 让老用户能够安全登录，在不同设备继续舰队、策略、积分和回放记录。
- 将登录自然接入“创建舰队 → 优化策略 → 开战”的单一任务链路。
- 同时支持简体中文和英文，并保持 Orbit Wars 原创星际战争视觉。
- 在真实认证启用后彻底移除公开环境的共享预览身份。

## 首发登录决定（2026-07-21）

- 首发只展示并开放 GitHub 登录；首次 GitHub 授权自动注册，之后直接登录同一账号。
- 邮箱密码、验证码与重置能力保留为关闭的 feature flag，不在页面展示，API 默认拒绝调用。
- Google OAuth 暂不纳入首发。

## 核心体验

### 全局账户入口

- 未登录时，页头在语言切换旁显示一个克制的“登录 / Sign in”入口。
- 已登录时，同一位置显示账户标识和简短菜单，包含当前邮箱或显示名、进入指挥中心、账户设置和退出。
- 页头仍只保留一个账户入口，不增加重复“立即开始”或任务菜单。

### 登录与注册弹窗

- 登录入口在当前页面打开轻量弹窗，不再进入独立认证页或全屏身份面板。
- 弹窗参考“韭见”的克制结构，但保持 Orbit Wars 的原创深色视觉；只显示“登录”标题、一句 GitHub 提示、GitHub 按钮、首次登录自动建档说明和明确关闭按钮，不承载产品介绍或功能卖点。
- 遮罩点击与 Escape 可以关闭，键盘焦点留在弹窗内，移动端不横向溢出。
- 登录或注册成功后返回触发认证的原始任务，例如创建舰队、策略实验室或开战。
- 旧 `/zh/auth`、`/en/auth` 链接兼容跳回 `returnTo` 页面并自动打开同一弹窗，不继续维护第二套登录界面。

### 后续邮箱能力（默认关闭）

- 邮箱注册、登录和找回密码不在首发页面展示，对应 API 默认返回不可用。
- 未来启用时继续采用一次性验证码、统一反馈和旧会话撤销。

### 会话与退出

- 登录状态默认保持 30 天；用户主动退出后当前会话立即失效。
- 受保护请求失效时，页面保留当前任务上下文并引导重新登录，而不是静默变成另一个用户。
- 用户数据按唯一账户隔离，一名用户仍只拥有一支主舰队。

### 身份提供方

- GitHub 是首发唯一注册与登录方式，配置不完整时入口保持禁用并显示上线前置。
- 不因 Email 或其他 provider 返回相同邮箱就自动合并账户。
- 已登录后的显式身份绑定与解绑不在第一阶段范围内。

## 安全与防滥用

- 公开真实认证只允许在域名和 HTTPS 下启用；IP + HTTP 预览继续使用无密码测试模式，但不得再伪装成真实账户能力。
- 密码长度为 8–128 位，服务端只保存带随机盐的强密码哈希，不保存明文或可逆密码。
- 注册验证码 10 分钟有效，60 秒内不可重复发送，15 分钟窗口最多发送 5 次，单个验证码最多尝试 5 次。
- 同一账户连续登录失败 5 次后短时锁定；错误信息不区分邮箱不存在和密码错误。
- OAuth 回调必须验证 state 和站内返回路径；认证写请求校验同源 Origin，并提供稳定错误 code。
- 会话使用 HttpOnly、SameSite Cookie；生产环境必须启用 Secure。

## 数据边界

- 真实注册用户获得独立账户、舰队、策略版本、Agent Key、积分和对局历史。
- 预热 Agent 继续作为系统舰队留在公开匹配池，不归属真实用户。
- `preview-commander` 的测试数据不自动迁移给任何注册用户。
- 第一阶段不提供删除账户、导出数据、管理员后台和社交身份合并，但数据模型不得阻塞后续补充。

## 范围

### 第一阶段包含

- GitHub OAuth 自动注册、登录、返回原任务和退出。
- 邮箱认证基础仅作为关闭的后续能力保留。
- 登录后返回原任务。
- 全局账户入口与账户菜单。
- 真实多用户会话与现有舰队/策略/比赛数据隔离。
- 中英文文案、移动端、键盘操作、错误态和加载态。
- 关闭公开共享预览身份的生产部署开关。

### 第一阶段不包含

- 手机号登录、短信验证码、Passkey、双因素认证。
- 自动合并不同登录渠道。
- 账户删除、数据导出、管理员用户管理。
- 好友、私信、社交主页或多人组队。

## 上线前提

- 配置正式域名与 HTTPS。
- 为 GitHub 配置准确的正式 HTTPS 回调地址；未配置时入口保持禁用。
- 生产环境关闭 `ORBIT_DEV_AUTH` 和 `NEXT_PUBLIC_ORBIT_DEV_SUBJECT`。

## 验收标准

- 新用户可以使用 GitHub 完成自动注册并建立独立舰队。
- 已注册用户可以使用同一 GitHub 身份登录，刷新与跨页面后会话仍有效。
- GitHub 入口只在配置完成时可点击，回调后返回原任务。
- 两个不同账户不能读取或修改对方舰队、策略、Agent Key 或私有模拟。
- 未登录访问受保护动作时进入认证流程，成功后准确返回原路径。
- 页头在未登录和已登录状态下只有一个清晰账户入口。
- GitHub 登录、自动注册和退出完整支持中英文、窄屏、键盘和可读错误反馈。
- 生产部署在域名 + HTTPS 下运行，且不再向浏览器注入固定预览身份。
