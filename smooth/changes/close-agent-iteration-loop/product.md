# 产品需求

## 背景

Orbit Wars 已经具备舰队创建、Agent Key、策略版本、真实匹配、公开历史和权威回放，但这些能力仍像分散页面。新用户不容易判断当前应该登录、建舰队、接入 Agent，还是直接开战；首页也仍以装饰性雷达证明“战斗”，回放结束后缺少明确的 Agent 迭代入口。

## 目标

- 全站只突出一个根据用户状态变化的“下一任务”主行动。
- 首页用真实公开对局和真实回放帧证明平台正在运行，不再依赖静态演示雷达。
- 回放页提供一键交给 Agent 分析的安全交接包，让用户自然进入下一轮策略迭代。
- 保持 Agent-only 主循环、中英双语、少量一级导航和原创星际战争表达。

## 核心体验

### 状态感知的下一任务

页头主行动按状态依次变化：

1. 未登录：开始游戏，进入登录与创建流程。
2. 已登录但未建舰队：创建舰队。
3. 已建舰队但没有有效 Agent Key：连接 Agent，进入指挥中心生成 Key。
4. 已有 Key 但没有当前 ready 策略：部署策略，进入指挥中心确认版本。
5. 舰队、Key 和当前策略均就绪：立即开战，进入竞技场。

页头不再同时展示“我的舰队 + 开战 + 退出”三组竞争行动；退出保留在任务菜单内。

### 首页真实战斗

- 首屏右侧展示最近公开对局的真实战场帧、双方舰队、当前 step 和结果，并自动短循环。
- reduced-motion 下停在一帧，不自动播放。
- 数据不可用时保留轻量战术占位，不阻塞标题、主行动和其他首页场景。
- 第三幕展示最近三场真实公开对局，并可直接进入永久回放；预热 Agent 的静态数字降为辅助信息。

### 回放交给 Agent

- 回放结果区增加“交给 Agent 分析”主按钮，一次复制结构化分析指令、公开回放链接、公开 compact 数据链接、双方、结果、评分变化、事实和关键事件。
- 同时提供“复制回放链接”和“复制 Agent 数据链接”两个明确动作。
- 复制后显示短暂的中英文成功反馈。
- 交接包不得包含 Agent Key、登录 Cookie、私有接口或任何只能由当前用户访问的数据。

## 范围

### 本轮包含

- 页头状态查询与下一任务决策。
- 首页真实公开对局列表和单场真实回放预览。
- 回放分析交接包及复制反馈。
- 所有新增文案和状态的中英文版本。
- 无数据、请求失败、窄屏、reduced-motion 和复制失败的降级。

### 本轮不包含

- Human 手动玩法。
- 新后端比赛协议、视频导出、社区评论、通知、杯赛或商店。
- 在首页加载完整回放的全部分段。
- 在网页内直接调用第三方 Agent 或上传用户密钥。
- 新增一级导航或第五个首页场景。

## 验收标准

- 未登录、无舰队、无有效 Key、无当前 ready 策略和完全就绪五种状态均有唯一且正确的页头主行动。
- 首页能从公开接口展示最近三场对局；存在回放时首屏至少渲染一个真实权威帧并链接到对应回放。
- 首页预览只读取 compact 与首个分段，不下载整场回放；失败或无数据时主页面仍完整可用。
- reduced-motion 下首页真实预览不自动轮播帧。
- 回放页三个复制动作可用，成功和失败都有可读反馈。
- Agent 分析包不包含密钥或私有数据，并提供可公开读取的 compact URL。
- `/zh` 与 `/en` 在桌面和窄屏无明显溢出、遮挡或未翻译主文案。
- 自动化测试、lint、类型检查、生产构建和浏览器验收通过。
