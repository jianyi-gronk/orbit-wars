# 产品需求

本阶段是 Human 指挥重新开放的第一阶段，先交付“手动训练 Beta”。整体目标是让玩家既能亲自下达战术命令，也能让平台策略或外部 Agent 自主作战；后续阶段再将验证稳定的手动玩法接入统一排位、人类对战和人机协同。

## 背景

Orbit/Wars 已有隐藏的 Human 实时链路：玩家和 Agent 使用同一 Observation、同一发射指令格式与同一引擎规则，服务端可签发比赛票据、通过实时连接接收命令、记录权威回放并标记控制方式。当前公开入口由 feature flag 关闭。

直接开放现有入口仍有明显上手障碍：玩家需要理解角度、兵力和每回合提交；默认回合等待时间会让上百步比赛过长；未提交空动作与真实断线没有被清晰区分；战术台也缺少目标预览、命令编辑和首次教学。

## 目标

- 不写代码、不连接外部 Agent 的用户可以在创建舰队后选择“亲自指挥”，完成一场可理解的训练赛。
- 保持 Human 与 Agent 的规则、信息、行动频率和指令上限一致，不给任一控制方式额外数值加成。
- 将手动操作从“填写角度参数”转化为直接操纵战场：选源星、指向目标、分配兵力、确认航线。
- 让一场典型手动训练控制在约 5～8 分钟，并能安全处理无操作、短暂断网和主动退出。
- 所有对局明确显示 `HUMAN` 或 `AGENT`，未来开放排位时继续使用唯一 rating 和统一榜单，只做控制方式标识与筛选。

## 产品结构

创建舰队后提供三个彼此平级、但主次清晰的行动：

1. **亲自指挥**：进入手动训练，推荐给想先理解规则的新用户。
2. **策略出战**：使用当前 `ready` 版本自动比赛。
3. **优化策略**：进入站内策略实验室，通过调参、代码或 AI 副驾迭代版本。

首页仍维持单一主行动“开始游戏”，不为三种玩法增加新的一级导航。具体选择发生在创建完成页或竞技场内。

## 核心体验

### 首次指挥教学

首次进入 Human 模式时先完成一个约 60～90 秒的交互教学：

- 识别己方、敌方和中立星球，以及星球内部的当前兵力。
- 点选己方星球，再点击目标方向或直接拖出一条发射航线。
- 使用 25%、50%、75%、最大值快捷档或精确滑杆分配兵力。
- 把命令加入本回合队列，理解一回合最多六条命令。
- 看懂预计航迹、可能首先接触的星球和到达时间；预测只作辅助，不替玩家选择最优行动。
- 提交一次命令并观察权威服务器结算。

教学允许暂停和重试，不影响战绩。完成后默认进入对平台训练 Agent 的第一场手动训练。

### 实时战术台

战术台优先保证战场可读和操作速度：

- 战场保持正方形；星球内部显示兵力，不显示无意义的星球 ID。
- 点击己方星球后，高亮合法源星、可用兵力和当前排队兵力。
- 点击目标或拖拽方向即可生成命令，系统把目标位置转换为实际发射角度。
- 航迹预览显示方向、派出兵力、预计接触点和大致到达时间，并明确“最终碰撞由服务器裁定”。
- 命令队列允许查看、单条删除、修改兵力、全部清空；提交后显示服务器确认状态。
- 顶部持续显示当前 step、回合倒计时、连接状态和本场控制标签。
- 键盘和触控操作均可完成核心命令；减弱动态效果时保留静态航迹与状态变化。

### 回合节奏

- 手动训练以约 2.5 秒为目标回合窗口，具体数值由实际可玩性测试调整。
- 玩家无需每回合点击“空提交”；连接正常但没有命令时自动视为合法不行动。
- 已排队且确认的命令只提交到对应 step，过期命令不能误发到下一回合。
- Agent 在同一个 step 接收同样的私有观察并受相同的六条命令上限约束。
- 比赛不能暂停等待玩家思考；仅教学和本地练习可以暂停。

### 断线、退出与恢复

- 短暂断线时战场继续运行，页面自动重连并从最近权威 step 恢复；断线期间默认不行动。
- 训练赛允许主动退出，结果标记为未完成或弃权，不改变 rating。
- 连续失联超过明确宽限窗口后结束比赛，回放和结果说明使用可理解的“连接中断”，不暴露内部错误码。
- 页面刷新后，只要比赛票据仍有效，就可以返回同一场比赛；票据失效时回到竞技场并说明原因。

### 结果与复盘

手动训练结束后展示：

- 胜负、胜因、关键事件和完整回放。
- 本场 `HUMAN` 控制标识、有效命令数、空过回合和平均提交时间。
- “再来一场”“交给当前策略重赛”“优化策略”三个后续行动。
- 从手动回放进入策略实验室时，可以把本场关键决策作为 AI 分析上下文，但必须由用户明确触发并消耗已说明的额度。

## 统一竞技规则

- 舰队只有一个总 rating、段位和全服名次，不因 Human 或 Agent 拆榜。
- 对局、历史、回放、公开档案和榜单记录清晰显示本场控制方式；筛选只是观察维度，不形成另一套排名。
- 正式排位开始后不得中途在人类和 Agent 之间切换，确保责任归因清晰。
- Human 和 Agent 只能看到自己规则允许的私有观察，不能读取对方隐藏数据或未来状态。
- 第一阶段的 Human 比赛只开放训练，不改变 rating；满足完成率、延迟、断线率和时长目标后，再按同一规则开放排位。

## 分阶段边界

### 第一阶段：手动训练 Beta

- Human 对平台训练 Agent。
- 首次交互教学、战术台、实时命令、断线恢复、结果与回放。
- 创建完成页和竞技场入口，中英文界面。
- 公开历史可显示训练记录及 `HUMAN` 标签，但不进入精彩筛选的默认推荐，除非比赛完整结束且达到战况阈值。

### 后续阶段

- **统一排位**：Human 对 Agent 进入同一 rating 和榜单，只显示控制标签；正式启用前先确定网络延迟补偿与弃权规则。
- **Human 对 Human**：实时匹配或邀请房间，需要双方就绪、独立票据和开赛确认。
- **人机协同**：Agent 默认执行，Human 在训练中进行有限接管或批准建议；若未来进入排位，必须使用独立的 `ASSISTED` 归因标签，且开赛后规则不可变化。
- **移动端完整指挥**：在触控可用性和性能达到桌面标准后再开放移动排位。

## 本阶段不包含

- Human 排位、Human 对 Human 或中途切换控制者。
- 允许玩家暂停、回退或修改已被服务器接受的命令。
- 自动瞄准最优目标、自动推荐必胜行动或向 Human 暴露 Agent 不可见的信息。
- 4P 混战、组队、观众干预、语音指挥和控制器外设。
- 为 Human 建立独立榜单、独立 rating 或数值补偿。

## 验收标准

- 无 Agent Key 的新用户可以从创建完成页进入 Human 教学，并连续完成第一场手动训练。
- 首次测试用户能在不阅读外部文档的情况下完成“选择源星、指向目标、分配兵力、提交命令”。
- 正方形战场中的归属、星球内兵力、航迹、当前 step、剩余时间和连接状态始终清晰可见。
- 玩家可使用点击/拖拽而不是手工理解弧度值生成合法命令，并能编辑或删除待提交命令。
- 无命令的已连接回合自动合法空过，不被当成断线；过期命令不会提交到错误 step。
- 短暂断线或刷新后能够恢复到最新权威状态；超过宽限窗口才以连接中断结束。
- 手动训练不改变 rating，完整生成控制标签、命令记录、结果和公开回放。
- Agent 与 Human 使用相同 Observation、命令格式、每步频率和命令数量上限。
- 中英文桌面流程完整；窄屏至少能完成教学和训练，若设备不满足操作要求必须在开赛前明确提示，不能比赛中途才阻断。
- feature flag 仍可快速关闭 Human 入口，关闭时 Agent 与策略实验室主循环不受影响。

## 上线门槛

手动训练 Beta 达到以下目标后，才进入统一排位设计与开放：

- 完整比赛率不低于 90%。
- 非主动退出的连接中断率低于 3%。
- 典型比赛时长中位数位于 5～8 分钟。
- 首次教学完成率不低于 80%。
- 命令拒绝率、过期提交率和恢复失败率均可观测，并处于可接受水平。

## 成功指标

- 创建舰队用户进入 Human 教学和完成首场训练的比例。
- Human 用户随后进入策略实验室或策略出战的转化率。
- 平均比赛时长、每场有效命令数、空过率、主动退出率和断线率。
- 手动训练后的回放打开率、重赛率和第二日回访率。
