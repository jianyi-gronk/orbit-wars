# 产品需求

## 背景

规则引擎使用 `100×100` 正方形战场，但当前 Web 战场按容器宽高分别缩放坐标，导致回放地图被横向拉成长方形，距离、轨迹和太阳危险区的视觉比例失真。中央太阳同时使用规则整齐的多层实心圆与十二根刻度线，外观更像靶心而非恒星。

## 目标

- 回放和实时战场都使用统一比例映射 `100×100` 正方形坐标。
- 宽高不一致时将正方形战场居中，不拉伸游戏坐标。
- 回放页面的战场容器本身保持正方形，形成清晰的独立棋盘。
- 太阳改为小型高温核心、不规则日冕与柔和辉光。
- 半径 `10` 的致命区仅用轻量虚线环表示，不再填成大面积棕色圆盘。

## 验收标准

- 任意容器尺寸下，X/Y 使用同一缩放比例，战场可视区域宽高相等。
- 宽屏与高屏容器分别产生水平或垂直居中偏移。
- 指针瞄准使用与绘制相同的正方形坐标变换。
- 回放战场 CSS 使用 `aspect-ratio: 1`，移动端不恢复成长方形。
- 太阳核心明显小于碰撞半径，危险环仍与 `SUN_RADIUS = 10` 一致。
- Web 测试、类型检查、lint 和生产构建通过。
