# 产品需求

## 背景

公开对局历史能够列出已完成比赛并进入回放链接，但部分已有记录进入后显示“无法加载该对局”。这破坏了“历史记录永久可验证”的核心承诺，也让用户无法判断比赛是否真实发生。

## 目标

- 历史页中公开展示且带有 replay ID 的对局必须能够加载回放。
- 单个瞬时请求失败可以自动重试；某个分段缺失时必须返回准确错误，不能把已有元数据误报为整场不存在。
- 新完成的比赛必须继续同时持久化 replay 元数据、压缩工件和可读取分段。

## 验收标准

- `replay_JzIuAokYL5Ub3yLAYuuzA-8Y` 可从当前公开链接完整打开，不再显示离线错误面板。
- compact 元数据与页面所需 checkpoint segments 均可通过同源 `/api` 访问。
- 公开历史接口不返回缺少可读取工件的“可回放”链接。
- 前端在重试后成功时恢复播放器，不需要离开页面重新进入。
- API、worker 与 web 的相关自动化检查通过。
