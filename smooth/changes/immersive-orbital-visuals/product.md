# 产品需求

## 背景

Orbit/Wars 首页已经能说明 Agent 对战流程，也具备滚轮分幕交互，但视觉仍像多张独立的产品说明页。用户需要在进入网站的第一秒感受到一个持续运行的轨道战场，并在下滑时看到同一个世界随任务阶段推进，而不是只看到界面模块切换。

## 目标

- 建立原创、鲜明的星际战争氛围，使首页第一眼更像游戏入口而非后台或普通 SaaS 落地页。
- 让四幕首页共享同一个连续世界：下滑时上一幕退场、下一幕登场，同时环境状态发生可感知的变化。
- 借鉴实时 HTML 与三维画面融合的层次感，但不复制参考站的布局、资产或标志性表达。
- 保持“部署第一支舰队”作为首屏唯一主行动，并继续支持简体中文和英文。

## 核心体验

### 首次进入

用户进入首页后应立即看到一个有纵深、有运动感的轨道战场。标题、说明与主按钮保持清晰可读，背景不能喧宾夺主。

### 分幕推进

用户通过滚轮、触控、方向键或场景导航推进四幕：

1. 接触：聚焦星球与战术扫描，强调部署入口。
2. 作战循环：环境展开为舰队部署与策略迭代路径。
3. 竞技网络：出现更多对手信标与连接关系，强调统一匹配池。
4. Agent 上行：环境收束为锁定目标与最终行动，推动用户开始部署。

切幕应有短促、连贯的空间变化和点击反馈，不能用长动画拖慢操作。

### 降级体验

- 系统开启 reduced-motion 时，页面保持完整构图与切幕能力，但不持续漂移或强制播放大幅动画。
- 不支持增强画面的设备仍能看到现有星图背景、全部内容和全部操作。
- 移动端不得出现横向溢出、文字遮挡或因增强画面导致的明显卡顿。

## 范围

### 本轮包含

- 首页四幕共享的沉浸式轨道背景。
- 场景切换对世界状态的联动。
- 鼠标轻微视差与按钮点击的短促游戏化反馈。
- 中英文、键盘、触控、reduced-motion 和无增强画面的兼容。
- 首页加载与渲染性能保护。

### 本轮不包含

- 重做战斗回放画面、竞技场、榜单或部署流程。
- 引入 Human 手动玩法入口。
- 使用外部星球大战 IP 素材、角色、音乐、标识或直接复刻参考站。
- 把实验浏览器能力作为核心依赖。
- 新增一级导航栏目。

## 验收标准

- 首屏打开后，主标题和“部署第一支舰队 / Deploy your first fleet”无需滚动即可清晰可见并可点击。
- 四幕共享一个连续的轨道世界；切换场景时至少在镜头、构图、信标密度或色温上出现明确变化。
- 滚轮、场景按钮、方向键、Page Up/Down、Home/End 均可切幕。
- 中文与英文首页均无溢出、遮挡和不可读文本。
- Canvas 或增强渲染初始化失败时，首页内容、导航和现有 CSS 背景仍可使用。
- reduced-motion 下没有持续旋转、漂移或大幅镜头动画。
- 移动端限制视觉复杂度，首页无横向滚动条。
- 自动化检查、类型检查和生产构建通过。
