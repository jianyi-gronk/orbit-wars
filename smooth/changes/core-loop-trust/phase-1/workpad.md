## 验收标准

- training、strategy simulation、ranked 与未知 mode 均按权威值准确显示；未知值不回退为排位赛。
- 私人 strategy simulation 默认不进入任何公共历史或其他用户视角。
- 只有当前候选版本关联 simulation 为 `COMPLETED` 且验证通过时才能发布；提交时再次确认最新资格。
- simulation 在刷新、切语言、重返页面后可恢复，`QUEUED` 与 `RUNNING` 会继续更新。
- Arena 创建后直达具体比赛状态，可从 `QUEUED`、`RUNNING`、`FINISHED` 追踪到 replay，不经过 Agent Key。
- replay 进入 strategy lab 后可见来源战果、最小高光和返回原 replay 的入口。
- 中英文 start 页无明显混用；已有可用舰队显示“进入竞技场 / Enter Arena”。
- 未登录隐藏退出项；已登录显示“退出登录 / Sign out”。

## 验证

- 用 ranked、training、私人 strategy simulation、未知 mode 四类数据分别核对公开历史、个人记录、状态页和 replay 标签。
- 用两个用户身份验证私人 simulation 仅发起者可见，且不出现在公开历史、公共列表或搜索入口。
- 覆盖 simulation 的 `QUEUED`、`RUNNING`、`COMPLETED + 通过`、`COMPLETED + 未通过`、`FAILED`、未知/暂不可达状态，并在刷新与中英文切换后复核。
- 验证旧候选版本的成功 simulation 不能解锁新候选版本；发布动作遇到状态变化时被最新权威结果拦截。
- 走一场真实 Arena 比赛，记录创建成功、排队、运行、结束、replay 处理中和 replay 可用的连续证据。
- 模拟一次状态请求瞬时失败，确认可手动或自动恢复且不会丢失比赛标识。
- 从有效、无效、无权限 replay 分别进入 strategy lab，核对上下文、隐私和返回目标。
- 在中文、英文及现有桌面/移动布局下检查 start 页、菜单、长状态文案和来源卡片。

## 备注

- 本阶段是克制的可信度修复，不包含代码或技术方案。
- “验证通过”沿用现有单场验证规则；不在 Phase 1 定义多场胜率门槛。
- 私人 simulation 可在发起者自己的策略记录中恢复，但默认不公开，也不提供分享开关。
- replay 高光复用已有摘要或事件；若不存在，以终局战果摘要作为最小高光，不新增识别算法。
- 多舰队、付费、移动导航重构和 Command Center 接入流程重构均不在本期。

## 疑问

- **未来私人模拟分享：** 分享的是 replay、结果还是策略？推荐先探索可撤销的只读链接，且分享不自动变更公开性。
- **未来多场胜率发布门槛：** 最少场次、对手覆盖和统计稳定性如何定义？需先积累数据，本期不设复杂阈值。
- **已有舰队首页 CTA：** Phase 1 固定为“进入竞技场”；后续是否根据最近任务在“继续上场”和“优化策略”间动态调整，需要用户行为验证。
- **多舰队：** 默认舰队、参赛资格、策略归属与跨页面切换上下文尚未定义，进入相关阶段前需独立讨论。
- **术语风格：** Phase 1 采用直接功能术语；未来若品牌化，需要决定世界观名称与清晰功能副标题的共存方式。
