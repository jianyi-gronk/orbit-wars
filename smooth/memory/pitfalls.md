# 对话踩坑记录

日常对话和分析中反复出现的问题。

## 待处理踩坑
- 从多选模式收敛为单一模式时，不能只隐藏一个选项并沿用原有卡片网格；必须重新设计为只读状态表达，并在中英文窄屏做视觉回归，检查字号、留白和动态长文本溢出。
- 导入 Kaggle 等第三方 Agent 时，固定验证赛通过仍不足以证明站内闭环；必须再走一次真实 worker 对局，特别核对 observation schema、角度范围、指令数量和锁定版本是否真的被执行。
- 预热 Agent 的舰队、比赛和 rating 结算成功，不等于公开对局闭环完成；必须同时核对 ReplayArtifact、Match.replay_id、对象存储工件、公开历史 API 和至少一个 replay segment，不能只看 finished 数或榜单场次。
- 样式文件“存在”不等于真实路由已加载；共享组件被双语正式路由使用时，必须从 RootLayout/实际路由核对 CSS chunk，并用密集真实数据检查绝对定位文本重叠，不能只停在源码中看到 CSS。
- 验收浏览器数据页时，“页面 200 + 直连 API 200”不足以证明用户旅程可用；必须走浏览器实际同源代理、检查每个渐进 segment，并验证瞬时失败后能自动/手动恢复，不能让一次 GET 失败永久锁死客户端页面。
- 回放协议不能只按 checkpoint/delta 建模；末段还可能包含无 frame 的 result 等终局元数据。消费者必须显式按 record type 分支，完整回归需验证最后 checkpoint 与 compact.frameCount 一致。
- 展示字体的 `line-height` 在英文中看似可用，不代表中文也安全；大字号中文低于约 `0.95` 时容易发生上下行字形碰撞。调整后还要复查长英文和窄屏是否产生意外第三行，必要时把语义行拆成独立 span 并分别约束。
- 页面使用内层滚动容器并把页头做成其兄弟节点时，滚轮监听不能只绑内层容器；鼠标停在固定页头、导航或菜单上会完全收不到事件，而外层 `overflow: hidden` 会让用户感知为“下滑消失”。应在页面生命周期内监听窗口，并实测内容区、页头区、连续滚动与反向滚动。
- “一次滚轮手势最多切换一幕”不能直接用过长的统一冷却实现；意图累计窗口和触发后的手势释放间隔应分离。释放间隔过长且被惯性持续延长时，会让用户进入第二幕后感觉后续滑动无响应。
- 累计式滚轮意图在方向反转时不能继续把正负增量相加；否则用户上滑返回前需要先抵消残留的下滑量，体感会明显迟钝。检测到反向输入时应按新方向重新累计。
