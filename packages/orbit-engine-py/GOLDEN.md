# Golden regression suite

The golden suite proves that `orbit-wars-2p-v1` preserves the audited Kaggle
Orbit Wars behavior without loading Kaggle Environments in production or CI.

## Coverage

| Case                          | Source                                   | Frames | Purpose                                             |
| ----------------------------- | ---------------------------------------- | -----: | --------------------------------------------------- |
| `fixed-seed-1234567-empty-55` | pinned oracle                            |     56 | deterministic reset/steps and the first comet spawn |
| `history-79969550`            | `notes/replays/2026-06-15/79969550.json` |     76 | real two-player command history and terminal result |
| `v69-vs-exp34-seed6`          | locally generated by pinned oracle       |    104 | representative producer v69 strategy match          |

Each frame records independent SHA-256 fingerprints for `planets` and
`fleets`, plus rewards, terminal status, counts, and the full authoritative
state hash. Tests replay the original actions twice and report a mismatch as
`case + step + field`.

Selected upstream regressions also cover map symmetry, swept moving-planet
collision, combat capture, and step-limit scoring.

## Floating-point policy

The oracle fixtures were generated on Darwin arm64 with Python 3.11.5. Physics
values are normalized to nine decimal places before hashing, establishing an
absolute comparison tolerance of `1e-9`. Integer identity, ownership, ships,
production, rewards, terminal step, and array ordering remain exact.

This normalization avoids insignificant `libm` differences while remaining
far below any gameplay-scale coordinate or collision threshold. A tolerance
change changes state hashes and therefore requires explicit review.

## Regeneration

Only regenerate against the audited Kaggle Environments 1.30.1 commit
`462efa26dd3d11018cde2b9e9ce9245b91cef471`:

```sh
/path/to/orbit-wars/.venv/bin/python \
  packages/orbit-engine-py/tools/generate_golden_fixtures.py \
  --orbit-project /path/to/orbit-wars \
  --output-dir /tmp/orbit-wars-goldens
```

Review metadata, action streams, frame counts, terminal steps, rewards, and
fixture diffs before replacing committed files. A rules behavior change must
create a new `ruleset_id`, not refresh these fixtures in place.
